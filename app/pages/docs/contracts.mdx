## Lobby

Entry point for players. Collects fees, manages countdown, groups players into railcars.

### Key Functions

```solidity
// Join the current game (send entryFee as value)
function joinGame() public payable

// Check if game can start (countdown expired or player limit hit)
function canStart() public view returns (bool)

// Current game ID
function currentGame() public view returns (uint256)

// Entry fee in wei
function entryFee() public view returns (uint256)

// Players in a specific game
function playerCount(uint256 gameID) public view returns (uint256)
```

### Admin

```solidity
function setEntryFee(uint256 newEntryFee) public          // DEFAULT_ADMIN_ROLE
function setPlayerLimit(uint256 limit) public              // DEFAULT_ADMIN_ROLE
function setTimeLimitToJoin(uint256 timeInSeconds) public  // DEFAULT_ADMIN_ROLE
```

## GameRound

Manages a single round: question delivery, answer submission, reveal, and scoring. There are 4 instances (round1-4), each linked to a question pack.

### Phases

```solidity
enum GamePhase { Pregame, Question, Reveal, Completed }
```

### Player Functions

```solidity
// Submit hashed answer (Question phase only)
function submitAnswers(bytes32 _hashedAnswer, uint256 gameID) public

// Reveal answers (Reveal phase only)
function revealAnswers(
    string questionAnswer,
    string crowdAnswer,
    string secretPhrase,
    uint256 gameID
) public
```

### Read Functions

```solidity
function phase(uint256 gameID) public view returns (uint8)
function getQuestion(uint256 gameID) public view returns (string, string[4])
function getResponseScores(uint256 gameID) public view returns (uint256[4])
function totalResponses(uint256 gameID) public view returns (uint256)
function totalReveals(uint256 gameID) public view returns (uint256)
function isMinorityRound(uint256 gameID) public view returns (bool)
function roundTimeLimit() public view returns (uint256)
```

### Scoring

| Action | Points |
|--------|--------|
| `submissionPoints` | 100 |
| `winningPoints` | 3,000 |
| `maxFastRevealBonus` | 1,000 |

Fast reveal bonus scales linearly: first to reveal gets the full 1,000 points, last gets 0.

## HjivemindKeeper

AutoLoop-compatible automation contract. Monitors all game rounds and advances state when conditions are met.

### AutoLoop Interface

```solidity
// Called by AutoLoop workers to check for pending work
function shouldProgressLoop() external view returns (bool loopIsReady, bytes memory progressWithData)

// Called by AutoLoop workers to execute pending work
function progressLoop(bytes calldata progressWithData) external
```

### VRF Integration

```solidity
// Register a controller's secp256k1 public key for VRF proofs
function registerControllerKey(address controller, uint256 pkX, uint256 pkY) external

// Enable/disable VRF processing
function setVRFEnabled(bool enabled) external

// Check VRF status
function vrfEnabled() public view returns (bool)
```

### Queue System

The keeper processes 6 queues in priority order:

1. **Lobby** - Start games when conditions met
2. **Round 1-4** - Advance round phases, deliver VRF seeds
3. **Winners** - Calculate final rankings

```solidity
function addActionToQueue(Action actionType, Queue queueType, uint256 gameID) public
```

Actions: `StartGame`, `StartRound`, `UpdatePhase`, `Clean`, `FindWinners`, `SeedRound`

## Access Control

All contracts use OpenZeppelin `AccessControlEnumerable`. Key roles:

| Role | Held By | Purpose |
|------|---------|---------|
| `DEFAULT_ADMIN_ROLE` | Admin wallet | Configuration, role grants |
| `SCORE_SETTER_ROLE` | Lobby, Rounds, Winners, GameController | Modify scores |
| `KEEPER_ROLE` | HjivemindKeeper, Admin | Advance round phases |
| `QUEUE_ROLE` | Lobby, Rounds, Winners | Add to keeper queue |
| `EVENT_SENDER_ROLE` | Rounds, Winners | Emit events on GameController |
| `GAME_ROUND_ROLE` | Round contracts | Query question packs |
